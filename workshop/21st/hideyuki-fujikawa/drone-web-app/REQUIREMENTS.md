# 要件定義書 (REQUIREMENTS)

## 概要

既存の CLI ベースのドローン制御体験を、Web ブラウザから操作できるように
置き換えるアプリケーション。ユーザーはブラウザ上でドローンへ接続し、状態を
リアルタイムに確認しながら各種コマンドを実行できる。

## 目的

- 専門的な CLI 操作を必要とせず、GUI でドローンを制御できるようにする。
- 機体の状態（位置・モード・アーム状態など）を視覚的・即時に把握できるようにする。
- ローカル環境（SITL を含む）で完結する最小構成で提供する。

## ターゲットユーザー

- ドローン開発者・学習者（ArduPilot / SITL を用いた検証を行う人）
- 飛行前後の動作確認を GUI で手軽に行いたいオペレーター

## 機能要件

### コマンド（WebSocket 経由）

| `type` | 挙動 | 入力 |
| --- | --- | --- |
| `connect` | 未接続なら機体への接続を開始（接続ボタン契機） | — |
| `arm` | `MAV_CMD_COMPONENT_ARM_DISARM`（`param1=1`） | — |
| `disarm` | `MAV_CMD_COMPONENT_ARM_DISARM`（`param1=0`） | — |
| `takeoff` | `MAV_CMD_NAV_TAKEOFF`（`param7=目標高度`） | 目標高度 |
| `land` | `MAV_CMD_NAV_LAND` | — |
| `goto` | `set_position_target_global_int_send` で目標位置送信 | 緯度・経度・高度 |
| `mode` | `set_mode()` でモード変更 | モード名 |

- `takeoff` / `goto` は実行前に `GUIDED` への切替を試みる（最大 5 秒待機）。
- モード変更は `set_mode()` を使用（`MAV_CMD_DO_SET_MODE` は使わない）。

### リアルタイム状態

| 項目 | キー | 表示形式 |
| --- | --- | --- |
| 接続状態 | `connected` | — |
| アーム状態 | `armed` | — |
| フライトモード | `mode` | — |
| 緯度 | `latitude` | 小数 6 桁 |
| 経度 | `longitude` | 小数 6 桁 |
| 高度 | `altitude` | 小数 2 桁 |
| ヘディング | `heading` | 整数 |

- 初期値: 接続/アーム = false、モード = `UNKNOWN`、数値 = 0。

### 地図

- Leaflet で表示。初期中心は東京駅付近 `35.681236, 139.767125`。
- 機体位置をマーカーで表示し、更新のたびに移動・地図中心を追従。
- 飛行軌跡をポリラインで表示し、WebSocket 再接続時にクリア。
- マーカーのポップアップに緯度・経度・高度を表示。

## 非機能要件

- **応答性**: MAVLink 受信のブロッキング処理を executor で実行し、
  イベントループ／WebSocket を停止させない。
- **堅牢性**:
  - 未接続時に制御コマンドを送っても致命的に落ちない。
  - WebSocket 切断時は受信タスクを安全に停止し、MAVLink 受信の例外はログを残す。
  - HEARTBEAT は発生源を選別し、GCS 等によるチラつきを防止する。
- **疎結合**: 状態更新はテレメトリー受信ベースで反映し、コマンド送信の成否を
  即断しない。
- **ポータビリティ**: ビルドツール不要の素の HTML/JS/CSS。Python 3.7 以上で動作。
- **レスポンシブ**: スマートフォン幅では縦並びに崩れる UI とする。

## 将来拡張案

- 接続先（接続文字列）を UI から変更可能にする。
- バッテリー残量・GPS 衛星数・速度などテレメトリー項目の追加。
- ミッション（ウェイポイント）の作成・アップロード・実行。
- 複数機体の同時監視・制御。
- 操作ログの保存・再生、フライトログのダウンロード。
- 認証・権限管理によるリモート運用対応。
