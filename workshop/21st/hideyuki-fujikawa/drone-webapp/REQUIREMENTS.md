# Drone Web App 要件定義

## 概要

CLI ベースのドローン制御体験を Web ブラウザに置き換える、ローカル動作の
ドローン Web 制御アプリケーション。バックエンド（FastAPI + pymavlink）と
フロントエンド（素の HTML / JavaScript / CSS）で構成する。

## 目的

- ブラウザから機体へ接続し、状態をリアルタイムに確認しながら基本操作
  （アーム・離陸・着陸・モード変更・指定座標への移動）を実行できるようにする
- ドローン制御（MAVLink / GCS 的な操作）の学習教材として、理解しやすい
  最小構成のリファレンス実装を提供する

## ターゲットユーザー

- ドローンスクール受講生などの、MAVLink / ArduPilot を学習中のエンジニア
- SITL を使ってドローン制御アプリの仕組みを学びたい開発者

## 機能要件

### 機体制御（WebSocket 経由の JSON コマンド）

| コマンド | 内容 |
| --- | --- |
| `connect` | 未接続なら機体（既定 `tcp:127.0.0.1:5762`）へ接続。接続ボタン契機 |
| `arm` / `disarm` | `MAV_CMD_COMPONENT_ARM_DISARM`（param1=1/0）を送信 |
| `takeoff` | GUIDED 切替（最大5秒待機）後、`MAV_CMD_NAV_TAKEOFF`（param7=目標高度） |
| `land` | `MAV_CMD_NAV_LAND` を送信 |
| `goto` | GUIDED 切替後、`set_position_target_global_int_send` で目標位置を送信 |
| `mode` | `set_mode()` でモード変更（GUIDED / AUTO / RTL / LOITER / STABILIZE） |

### リアルタイム状態表示

- 接続状態・アーム状態・フライトモード・緯度（小数6桁）・経度（小数6桁）・
  高度（小数2桁）・ヘディング（整数）を表示する
- 状態は MAVLink テレメトリー（`HEARTBEAT` / `GLOBAL_POSITION_INT`）の
  受信ベースで更新し、コマンド送信の成否では即断しない
- HEARTBEAT は GCS など機体以外の発生源を除外し、オートパイロットのみ採用する

### 地図表示

- Leaflet + OpenStreetMap タイル。初期中心は東京駅付近（35.681236, 139.767125）
- 機体位置マーカーを表示し、位置更新のたびに移動・地図中心を追従する
- 飛行軌跡をポリラインで描画し、WebSocket 再接続時にクリアする
- マーカーのポップアップに緯度・経度・高度を表示する

### WebSocket

- `WebSocket /ws` を提供し、接続直後に現在の状態を即時送信する
- コマンド結果は `{"type":"status","message":"..."}`、状態更新は
  `{"type":"state","state":{...}}` で送る
- クライアントは切断後 3 秒で自動再接続する

## 非機能要件

- 起動ポートは 9999。`http://127.0.0.1:9999/` で UI を表示できること
- MAVLink の受信・接続待ちで FastAPI のイベントループをブロックしないこと
  （executor 経由で実行する）
- 未接続時に制御コマンドを受けても致命的に落ちないこと
- MAVLink 受信の例外はログに残すこと
- スマートフォン幅でも UI が破綻しないレスポンシブ対応
- ビルドツール不要（フロントエンドは静的ファイルのみ）

## 将来拡張案

- 接続先（connection string）を UI から指定できるようにする
- 地図クリックで GoTo 目標地点を指定する
- ミッション（ウェイポイント）のアップロード・実行
- バッテリー残量・GPS 衛星数・対気/対地速度などの表示項目追加
- 複数機体の同時接続・切替
- 認証や HTTPS 対応などリモート運用向けのセキュリティ強化
