# 要件定義書 — Drone Web App

## 概要

既存の CLI ベースのドローン制御体験を、Web ブラウザから操作できるように置き換える
最小構成のアプリケーション。FastAPI + WebSocket + pymavlink をバックエンド、素の
HTML/JS/CSS をフロントエンドとし、ローカル環境で動作する。

## 目的

- ブラウザからドローンへ接続し、状態をリアルタイムに把握できるようにする。
- アーム / 離陸 / 着陸 / モード変更 / 指定座標への移動を Web UI から実行できるようにする。
- CLI 操作に不慣れなユーザーでも直感的に機体を操作できるようにする。

## ターゲットユーザー

- ドローン開発・検証を行うエンジニア
- SITL（シミュレータ）で操作を学習する受講者
- CLI ではなく GUI で機体を操作したい運用者

## 機能要件

- **接続**: 接続ボタンで MAVLink 接続（`tcp:127.0.0.1:5762`）を開始する。起動時の自動接続は行わない。
- **アーム / ディスアーム**: `MAV_CMD_COMPONENT_ARM_DISARM` を送信する。
- **離陸**: 目標高度を指定して `MAV_CMD_NAV_TAKEOFF` を送信する（事前に GUIDED へ切替）。
- **着陸**: `MAV_CMD_NAV_LAND` を送信する。
- **GoTo**: 緯度・経度・高度を指定し `set_position_target_global_int` で移動する（事前に GUIDED へ切替）。
- **モード変更**: `set_mode()` で `GUIDED` / `AUTO` / `RTL` / `LOITER` / `STABILIZE` を切り替える。
- **状態表示**: 接続・アーム・モード・緯度・経度・高度・ヘディングをリアルタイム更新する。
- **地図表示**: 機体位置のマーカー、飛行軌跡ポリライン、地図中心の追従を行う。

## 非機能要件

- **応答性**: MAVLink の blocking 受信を executor 上で実行し、WebSocket / イベントループを
  ブロックしない。
- **堅牢性**: 未接続時に制御コマンドを送っても致命的に落ちない。受信例外はログに残す。
- **正確性**: 単位変換（緯度経度 1e7、高度 mm→m、ヘディング /100）を確実に行う。
- **信頼性**: SITL の GCS 等の HEARTBEAT を除外し、本物のオートパイロットのみで状態更新する。
- **可搬性**: フロントエンドはビルドツール不要。Python 3.7 以上で動作する。
- **レスポンシブ**: スマートフォン幅でも UI が破綻しない。

## 将来拡張案

- ミッション（ウェイポイント）の作成・アップロード・実行
- バッテリー残量・GPS Fix・衛星数などのテレメトリー拡充
- 複数機体の同時管理
- 操作ログ・フライトログの記録と再生
- 認証・アクセス制御による安全対策
- 接続先（シリアル / UDP / TCP）の UI からの切替
