# ドローンWeb制御アプリケーション - 詳細要件仕様書

## 概要

既存のCLIベースのドローン制御体験を、Webブラウザから操作できるように置き換えるプロジェクトです。ユーザーはブラウザ上でドローンへ接続し、状態をリアルタイムに確認しながら、アーム、離陸、着陸、モード変更、指定座標への移動を実行できます。

## 目的

- Webブラウザを通じたドローンの直感的な操作体験を提供
- リアルタイムな機体状態の可視化
- GCSなしでドローン制御を学習可能な環境構築

## ターゲットユーザー

- ドローンエンジニア養成塾の受講生
- ドローン制御を学ぶ学習者
- MavlinkとWeb技術の両立を学びたい開発者

## 機能要件

### 1. 接続管理

| 項目 | 要件 | 優先度 |
| --- | --- | --- |
| 自動接続 | ページロード時にWebSocketのみ接続、MAVLink接続は不自動 | 必須 |
| 機体接続 | 接続ボタン操作で MAVLink 接続開始 | 必須 |
| 接続状態表示 | リアルタイム表示、接続/未接続の2状態 | 必須 |
| 切断時の再接続 | WebSocket切断時、3秒後に自動再接続 | 必須 |

### 2. 機体制御コマンド

| コマンド | 入力 | 動作 | 優先度 |
| --- | --- | --- | --- |
| `arm` | — | MOTORを起動準備 | 必須 |
| `disarm` | — | MOTORを停止 | 必須 |
| `takeoff` | 目標高度(m) | GUIDED切り替え後、指定高度まで上昇 | 必須 |
| `land` | — | 着陸コマンド送信 | 必須 |
| `goto` | 緯度・経度・高度 | GUIDED切り替え後、指定地点へ移動 | 必須 |
| `mode` | モード名 | GUIDED/AUTO/RTL/LOITER/STABILIZEへ切り替え | 必須 |

### 3. リアルタイム状態表示

| 項目 | データソース | 形式 | 更新頻度 | 優先度 |
| --- | --- | --- | --- | --- |
| 接続状態 | MAVLink接続確認 | boolean | 接続時 | 必須 |
| アーム状態 | HEARTBEAT.base_mode | boolean | テレメトリーベース | 必須 |
| フライトモード | HEARTBEAT.custom_mode | 文字列 | テレメトリーベース | 必須 |
| 緯度 | GLOBAL_POSITION_INT.lat | 小数6桁 | テレメトリーベース | 必須 |
| 経度 | GLOBAL_POSITION_INT.lon | 小数6桁 | テレメトリーベース | 必須 |
| 高度 | GLOBAL_POSITION_INT.relative_alt | 小数2桁(m) | テレメトリーベース | 必須 |
| ヘディング | GLOBAL_POSITION_INT.hdg | 整数(°) | テレメトリーベース | 必須 |

### 4. 地図機能

| 項目 | 要件 | 優先度 |
| --- | --- | --- |
| 表示ライブラリ | Leaflet + OpenStreetMap タイル | 必須 |
| 初期中心 | 東京駅付近（35.681236, 139.767125） | 必須 |
| 機体マーカー | 位置更新のたびに移動 | 必須 |
| マーカーポップアップ | 緯度・経度・高度を表示 | 必須 |
| 飛行軌跡 | ポリラインで記録、WebSocket再接続時にクリア | 必須 |
| 自動スクロール | 機体位置を常に中心に表示 | 必須 |

### 5. UI/UX

| 項目 | 要件 | 優先度 |
| --- | --- | --- |
| レイアウト | 2カラム（パネル）+ 地図 | 必須 |
| パネル内容 | ステータス表示 + コントロール操作 | 必須 |
| ボタン配置 | 機能ごとにグループ化 | 必須 |
| レスポンシブ対応 | PC / タブレット / スマートフォン対応 | 必須 |
| メッセージ表示 | コマンド実行結果をリアルタイム表示 | 必須 |

## 非機能要件

| 項目 | 要件 | 優先度 |
| --- | --- | --- |
| 起動ポート | 9999 | 必須 |
| 接続先デフォルト | tcp:127.0.0.1:5762 | 必須 |
| 言語 | Python 3.7 + | 必須 |
| WebSocketエンドポイント | /ws | 必須 |
| 静的ファイル配信 | /static にマウント | 必須 |
| HEARTBEATフィルタリング | GCS型HEARTBEATを除外 | 必須 |
| イベントループブロック | MAVLink受信で非ブロッキング対応 | 必須 |
| エラーハンドリング | 致命的な落下防止、ログ記録 | 必須 |

## 技術制約

- バックエンド: Python + FastAPI, リアルタイム通信: WebSocket
- MAVLink通信: pymavlink
- フロントエンド: 素のHTML + JavaScript + CSS（ビルドツール不要）
- 地図表示: Leaflet + OpenStreetMap タイル
- 単一リポジトリ内に `backend` と `frontend` を分離

## 受け入れ条件（テスト項目）

1. ✅ サーバー起動後に `http://127.0.0.1:9999/` で UI が表示される
2. ✅ 接続ボタンで MAVLink 接続が始まる
3. ✅ アーム・ディスアーム・離陸・着陸・GoTo・モード変更の各コマンドを送れる
4. ✅ 状態（接続・アーム・モード・緯度・経度・高度・ヘディング）がリアルタイムに更新される
5. ✅ 地図上のマーカーと軌跡が更新される
6. ✅ モバイル幅でも UI が破綻しない
